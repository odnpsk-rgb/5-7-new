# RAG-ассистент по строительным нормативам

ИИ-ассистент с технологией RAG (поиск по базе знаний), локальным векторным хранилищем ChromaDB, кешированием ответов и Telegram-ботом.

## Бизнес-ценность

Ассистент ускоряет работу технической поддержки: ответы формируются на основе загруженных нормативных документов (СП), а кеш снижает время повторных ответов и расход API.

## Описание

1. **Пользователь задаёт вопрос**
2. **Проверка кеша** — если ответ уже есть, возвращается из кеша
3. **Поиск релевантных фрагментов** — векторный поиск в ChromaDB
4. **Формирование контекста** — сборка найденных фрагментов
5. **Генерация ответа** — запрос к OpenAI с контекстом
6. **Ссылки на источники** — в Telegram бот показывает, из каких документов взята информация
7. **Кеширование и логирование** — ответ сохраняется в кеш, запрос пишется в SQLite

## Особенности

- Локальное векторное хранилище ChromaDB
- OpenAI-эмбеддинги (`text-embedding-3-small`)
- База знаний из txt-файлов в папке `docs/` (своды правил по строительству)
- Кеширование ответов LLM
- Telegram-бот с демо-лимитом (4 сообщения на пользователя)
- Ссылки на источники в ответах бота
- Логирование запросов в `logs.db`
- Поддержка VPN/прокси (SOCKS4/SOCKS5, HTTP)
- Консольный режим: интерактивный и демонстрационный

## База знаний (`docs/`)

Документы для RAG хранятся как **txt-файлы** в папке `docs/`. При первом запуске, если ChromaDB пуста, они загружаются автоматически через `load_documents_from_folder()`.

Сейчас в базе:

| Файл | Документ |
|------|----------|
| `poli.txt` | СП 29.13330.2011 «Полы» |
| `organizacia stroitelstva.txt` | СП 48.13330.2011 «Организация строительства» |
| `ograzhdausie konstrukcii.txt` | СП 70.13330.2012 «Несущие и ограждающие конструкции» |

### Подготовка документов из PDF

```bash
python extract_docs.py
```

Скрипт читает все `*.pdf` в `docs/` и сохраняет очищенный текст в одноимённые `.txt`.

### Обновление базы после смены документов

1. Положите или обновите `.txt` в `docs/`
2. Удалите папку `chroma_db/`
3. Очистите кеш: команда `clear_cache` в консоли или удалите `cache.json`
4. Запустите `python main.py` — документы переиндексируются

## Запуск

```bash
python main.py
```

При первом запуске создаются `chroma_db/`, при необходимости индексируются документы из `docs/`.

### Режимы работы

**1. Интерактивный режим** (по умолчанию)  
Задавайте вопросы в консоли. Команды: `exit`, `cache`, `clear_cache`, `stats`, `logs`.

**2. Демонстрационный режим**  
Готовые примеры вопросов с демонстрацией RAG и кеша.

**3. Telegram-бот** (если задан `TELEGRAM_BOT_TOKEN`)  
- Ответы на основе базы знаний  
- После ответа — блок **«Источники»** с названием СП, файлом, релевантностью и цитатой  
- **Демо-лимит:** не более 4 сообщений на пользователя, далее: *«Демо-режим: 4 сообщения. Для открытия доступа свяжитесь с администратором.»*  
- Команды: `/start`, `/help`, `/stats`, `/logs`

## VPN и прокси

Если VPN выставляет системный прокси `socks4://...`, httpx по умолчанию падает с ошибкой. В проекте это обработано в `http_proxy.py` и `openai_client.py`: `socks4` приводится к `socks5`, для клиентов включён `trust_env=False`.

При проблемах с подключением укажите прокси явно в `.env`:

```env
OPENAI_PROXY=socks5://127.0.0.1:10808
TELEGRAM_PROXY=socks5://127.0.0.1:10808
```

Порт и тип прокси смотрите в настройках VPN-клиента (Clash, V2Ray и т.п.).

## Архитектура

### `embeddings.py`

- Разбивка текстов на чанки (500 символов, перекрытие 50)
- Эмбеддинги через OpenAI API
- Хранение и поиск в ChromaDB
- `load_documents_from_folder("docs")` — загрузка всех `.txt`

### `rag.py`

```python
assistant = RAGAssistant(embedding_store)
answer, search_results = assistant.generate_response(query, top_k=3)
sources = format_source_references(search_results)
```

### `telegram_bot.py`

- RAG + кеш + логирование
- Счётчик сообщений пользователя в `logs.db` (таблица `telegram_dialog_counts`)
- Источники отправляются отдельным сообщением после ответа

### `cache.py`

Ключ — SHA-256 нормализованного запроса. В кеш сохраняется только текст ответа LLM (без блока источников).

## Кастомизация

### Новые документы

1. Добавьте `.txt` в `docs/`
2. При необходимости допишите название в `DOCUMENT_TITLES` в `rag.py`
3. Пересоздайте `chroma_db/` и очистите кеш

### Модель LLM

В `main.py`:

```python
rag_assistant = RAGAssistant(
    embedding_store=embedding_store,
    model="gpt-3.5-turbo",
    temperature=0.7,
)
```

### Лимит сообщений в Telegram

В `telegram_bot.py`:

```python
DEMO_MESSAGE_LIMIT = 4
```

### Кликабельные ссылки на документы

Разместите txt-файлы по URL и укажите в `.env`:

```env
DOCS_BASE_URL=https://your-site.com/docs
```

|-----------|-----------|
| Эмбеддинги | `text-embedding-3-small`, 1536 измерений |
| ChromaDB | `./chroma_db/`, cosine similarity |

## Известные ограничения

1. Нужен **OPENAI_API_KEY** и интернет (эмбеддинги и генерация — через API).
2. **Кеш** не учитывает обновление документов — после смены `docs/` очистите кеш и `chroma_db/`.
3. **Демо-лимит** в Telegram сохраняется в БД и не сбрасывается при перезапуске бота.
4. Без **DOCS_BASE_URL** ссылки на документы в боте показываются как путь к файлу (`docs/...`), без кликабельного URL.
5. При медленном VPN возможны таймауты Telegram API — проверьте прокси и доступность `api.telegram.org`.
